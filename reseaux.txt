RESEAUX

Notions de bases :

Les réseaux sont souvent organisés selon un principe de clients et de serveurs.

Requête /  réponse -> le client et le serveur vont communiquer au moyen d'une requête et une réponse (on renvoi une info )

URL -> Composé de plusieurs éléments :
*   protocol
*   utilisateurs
*   domaines
*   ports
*   chemins
*   ressources
protocole: //utilisateur@domaine.fr:port/chemin/ressource.ressource
e.g : http://colombbus.org/contact/index.html




- Firefox est un client server
- Poste client et serveur 
HTTP : est un protocol réseau.
Il fonctionne avec un logiciel client (navigateur de recherche) et un serveur HTTP (Apache ou Nginx)
LARAGON - WAMPP :
Des sacs réunissant plusieurs logiciels pour simplifiier installation et utilisation de site
- Serveur HTTP (Apache / Nginx / Nodejs (serveur HTTP particulier))
- Serveur MySQL / client: PHPMyAdmin

PHP : est language informatique qui fonctionne à l'intérieur d'un serveur HTTP. PHP sert à générer les pages dynamiquement.
